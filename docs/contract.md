# Droidspaces initramfs / macOS app contract

## Kernel command line

Required:

```text
console=hvc0 init=/init
```

Recommended:

```text
droidspaces.share_tag=dsdata
droidspaces.config=/mnt/host/Config/desired-state.conf
```

Debug modes:

```text
droidspaces.mode=shell
droidspaces.mode=prepare-only
```

## Shared directory

The macOS app should expose one writable host directory through `VZSingleDirectoryShare` and tag it `dsdata` unless overridden.

Guest mount point:

```text
/mnt/host
```

Expected layout:

```text
RootfsTarballs/
Images/
Containers/
Config/
Cache/
Logs/
```

## Early image import format

The first implementation uses a simple line-oriented file generated by the macOS app from plist:

```text
image.debian13.tar=RootfsTarballs/debian13-aarch64.tar.zst
image.debian13.path=Images/debian13-aarch64.img
image.debian13.size=8G
image.debian13.fs=ext4
```

This is an intermediate guest-facing format. The macOS-facing configuration should remain plist-native.
